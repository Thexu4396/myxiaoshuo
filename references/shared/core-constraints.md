## 三大定律（低自由度 - 必须精确执行）

| 定律 | 规则 | 检查方式 |
|------|------|----------|
| **大纲即法律** | 严格执行大纲，不得擅自发挥 | 审查时对照大纲 |
| **设定即物理** | 实力/招式/物品 ≤ index.db 记录 | 写作前查询确认 |
| **发明需识别** | 新实体由 Data Agent 自动提取 | 章节完成后处理 |

## 新实体处理流程

当前规则：正文不再要求 XML 标签：

1. **写作时**: 直接写纯正文，新角色/地点/物品正常描写
2. **完成后**: Data Agent 自动识别新实体并写入 index.db
3. **不确定实体**: Data Agent 标记为 uncertain，由人工确认

## 章节约束分层

### Hard（必须）

- 正式正文创作前，必须先与用户沟通并确认详细大纲；未确认时只能输出大纲、提问或修订方案，不能直接写正文。
- 详细大纲至少包含：本章目标、必要信息、关系/局势变化、开场状态、分段推进、结尾钩子、伏笔回收/埋设、文风提醒和连贯性检查。
- 每章正式正文的字数必须控制在 2300-2700 字。
- 每章必须有明确主功能：推进剧情 / 揭露信息 / 关系变化 / 爽点兑现 / 危机升级中至少一项为主，禁止整章功能漂移。
- 本章可读性达标：读者能回答“发生了什么 / 谁在做什么 / 为什么”。
- 本章必须存在清晰推进：问题、目标、代价、关系变化、信息变化至少一项可被识别。
- 本章必须有可见变化：局势、人物态度、关系温度、已知信息、资源状态、风险等级中至少一项在章末与章初不同。
- 本章冲突和结果必须保持因果链清晰，禁止只给结果不给触发过程。
- 若上章有明确承诺（钩子/未闭合问题），本章必须回应（允许部分兑现，不要求一次性结清）。
- 本章结尾必须保留一个前拉力：新问题、新危机、新决定、新发现、未兑现承诺中至少一项仍然成立。
- 单章默认保持单一主视角；若确需切换视角，必须有明显分隔且切换后带来新增信息或戏剧价值。
- 每章正文写完后，必须进行一次“去 AI 味”检查与润色：删除总结腔、套话、重复句式、空泛情绪标签和机械过渡，让表达更接近自然人写作。
- 每章正文定稿后，必须自动回填并完善相关资料：更新章节小纲/章节记录，并把新增设定、人物状态变化、关系变化、伏笔回收或新增伏笔同步到对应设定文件或维护记录。
- 禁止输出占位正文（如 `[待补充]`、`[TODO]`、`...（省略）...`）。

### Soft（建议）

- 开头尽早进入冲突/风险/强情绪（建议前 200-400 字，不再固定 120 字）。
- 2300-2700 字章节建议采用“小三段”节奏：开场建立问题，中段升级冲突，结尾兑现部分结果并抛出新钩子。
- 字数分配可参考：开场 300-500 字，中段推进 1400-1800 字，结尾 300-500 字；允许按章型微调。
- 每章建议只承载 1 个主冲突和 1-2 个次冲突，避免短章里并行线过多导致发散。
- 未闭合问题或下一章期待锚点建议放在章末或后段（不限定“后 80-150 字”）。
- 局面变化建议保持节奏感（参考 800-1400 字一个脉冲，短章至少一次实质变化）。
- 微兑现频率按题材 profile 建议执行，不要求机械等间距。
- 过渡章允许爆点较轻，但仍建议给到读者至少一种“收获”：新信息、新判断、关系推进、小反打、小兑现之一。

### Style（可选强化）

- 正文默认采用“冷静观察型”语言：先写可见事实、动作、细节和压力，再让情绪从反应里浮出来，少做作者代言式判断。
- 场景描写优先服务冲突、人物状态和信息释放，避免与当前目标无关的空镜堆砌。
- 对话尽量带意图（试探/回避/施压/诱导），减少纯说明句。
- 避免连续大段纯解释；若必须解释，建议切分为”信息 + 行动/反应”。
- 心理活动应推动选择、误判或情绪升级，避免只复述读者已知结论。
- 避免”回去休息了”式机械收尾；若使用平缓收尾，需同时保留未闭合期待。

### Anti-AI（写作时预防）

- 禁止连续 3 段以上使用相同句式结构。
- 情绪描写必须通过行为/生理暗示，禁止直接标签（”他感到X”）。
- 每 500 字至少有一次节奏变化（短句爆发、对话插入、场景切换）。
- 对话必须带意图冲突，禁止纯信息传递。
- 禁止用“他想了想”“他明白了”“气氛一下子凝重起来”等空泛句直接代替过程描写。
- 章末禁止”安全着陆”——必须保留至少一个未解决的问题或不安感。
- 删掉万能副词（缓缓/淡淡/微微），用具体动作替代。
- 叙述避免大开大合的宣告腔，优先让读者从观察结果自己得出紧张、寒意或压迫感。

## 章节完成后维护

- 定稿后先做一轮人味化检查：重点检查句式重复、解释腔、总结腔、直白情绪标签、空泛转场和“像提纲扩写”的段落。
- 完成后同步维护相关文档：至少更新本章章纲结果、人物状态变化、关系温度变化、设定新增/修正、伏笔回收/新增中的相关项。
- 若本章实际发展与原大纲存在合理偏移，必须在不违背主线的前提下补正后续大纲或维护备注，避免后文脱节。

## 去 AI 味检查清单

- 句式检查：是否连续多段都在使用相同句式、相同节奏或“动作 + 解释”套版；若有，至少改出 2-3 处明显变化。
- 总结腔检查：删掉“他明白了”“这一刻他终于意识到”“气氛顿时如何”等总结句，改成动作、反应或对话落点。
- 情绪标签检查：把“愤怒、震惊、紧张、悲伤”这类直接标签，尽量改成生理反应、微动作、语言变化或错误决策。
- 副词套话检查：集中搜索“缓缓、淡淡、微微、轻轻、忽然、顿时”等高频词，能删则删，能换具体动作就不保留空泛修饰。
- 对话人味检查：对话是否过于完整、礼貌、轮流、像汇报；适当加入打断、省略、答非所问、潜台词和角色口癖。
- 节奏起伏检查：是否段落长度过于平均、信息密度过匀；至少保留一处短句爆点、一处留白或一处明显停顿。
- 解释冗余检查：凡是“展示完又解释一遍”的句子，优先删掉解释部分，信任读者理解。
- 提纲扩写感检查：如果某段只是把章纲直白展开，没有现场感、动作链和具体细节，必须补足感官、行为或临场反应。
- 章末收口检查：结尾是否写成平稳总结或机械收束；若是，改成带悬念、代价、决定或异样感的结束方式。
- 角色差异检查：主要角色的反应、措辞和动作是否能区分身份与性格，避免全员同一腔调。

## 单章交付前自检

- [ ] 字数是否在 2300-2700 字之间
- [ ] 本章主功能是否明确，且没有写成流水账
- [ ] 上章承诺或钩子是否得到回应
- [ ] 本章是否至少产生一项可见变化
- [ ] 冲突推进是否有明确因果，不靠作者硬推
- [ ] 章末是否保留下一章追读动力
- [ ] 是否存在设定冲突、人物失真或重复上一章功能
- [ ] 是否删掉占位句、总结腔和明显 AI 套话
- [ ] 是否已做去 AI 味润色，让措辞、节奏和细节更接近自然人写法
- [ ] 是否已回填并更新大纲、设定和相关维护记录

## 爽点与节奏（按题材 profile 调整）

- 爽点密度由题材与章型共同决定：过渡章允许低密度，但不允许“整章无收获”。
- 组合爽点与里程碑爽点采用滚动窗口评估（5章/10-15章），用于预警而非逐章硬判。
- 连续同类型爽点达到 3 章记为风险预警，优先通过类型变体或执行差异化修正。

## Strand 平衡警告

| 情节线 | 警告条件 |
|--------|----------|
| Quest（主线） | 连续 5+ 章 |
| Fire（感情线） | >10 章未出现 |
| Constellation（世界观） | >15 章未出现 |

## 长线维护

- 连续规划或写作 3 章以上时，必须检查 `大纲/长线追踪.md` 或等价维护表。
- 若本章改变了人物立场、关系温度、伏笔状态、世界规则或主线阶段，完成后必须记录为 canon patch 或维护备注。
- 若最近三章功能重复，下一章优先安排升级、反转、兑现、新信息、关系变化或代价落地。

## 禁止事项

- `[待补充]`、`[TODO]`、`...（省略）...` → 必须完整写出
- 战斗后无善后描述（都市异能题材）
